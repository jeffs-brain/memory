`jeffs-brain/memory` is a cross-language memory library for LLM agents, driven by a shared HTTP and storage specification so every SDK behaves the same. Local-first, hosted-optional, Apache-2.0.

Cross-SDK eval parity today is the shared daemon surface in three scenarios: `ask-basic`, `ask-augmented`, and `search-retrieve-only`. The shared runner defaults retrieval mode to `auto`, so each daemon resolves it locally. Full LongMemEval replay and agentic runs stay in the native SDK runners.

## Quick install

<Tabs>
  <TabItem label="TypeScript">
    ```bash
    npm i -g @jeffs-brain/memory
    memory init ./my-brain
    memory ingest ./docs
    memory search "question"
    ```
  </TabItem>
  <TabItem label="Go">
    ```bash
    go install github.com/jeffs-brain/memory/go/cmd/memory@latest
    memory init ./my-brain
    memory ingest ./docs
    memory search "question"
    ```
  </TabItem>
  <TabItem label="Python">
    ```bash
    pip install jeffs-brain-memory
    memory init ./my-brain
    memory ingest ./docs
    memory search "question"
    ```
  </TabItem>
</Tabs>

## MCP in four lines

```bash
claude mcp add jeffs-brain -- npx -y @jeffs-brain/memory-mcp
```

That registers a stdio MCP server with Claude Code. The same package works with Claude Desktop, Cursor, Windsurf, and Zed via the standard `mcpServers` JSON block.
