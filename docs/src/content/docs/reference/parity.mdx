## React Native surface

The React Native SDK is the mobile and client runtime surface. It shares the storage format, retrieval algorithms, memory pipeline semantics, ACL wrapper, and knowledge helpers, but it is not a daemon or CLI package.

- It targets Expo or React Native apps with a local file adapter and SQLite driver.
- Local on-device inference is supported through app-provided native inference and embedding modules.
- Cloud usage works through the built-in OpenAI-compatible provider and embedder, plus `HttpStore`.
- Some server-oriented helpers, CLI features, and eval tooling remain TypeScript, Go, or Python only.

## Postgres adapter is TypeScript-only

`PostgresStore` lives in `@jeffs-brain/memory-postgres` and is not ported to
Go or Python. If you need a Postgres-backed brain from a Go or Python
process, run a TypeScript daemon (`memory serve`) on top of `PostgresStore`
and point your Go or Python process at it through `HttpStore`.

That keeps you on the shared HTTP protocol surface only. It does not give Go
or Python the TypeScript-only local store APIs or index-management helpers.

## Memory stages are uneven across SDKs

All three SDKs implement the four stages (extract, reflect, recall, consolidate), the L0 buffer, the procedural detector, and the episode recorder. They diverge at the orchestration layer:

- TypeScript ships a full lifecycle wrapper that hosts the stages, the buffer, the procedural detector, and the episode recorder behind a single entry point.
- Go and Python expose each stage as a discrete function or struct. Callers wire them together themselves, or drive them through the HTTP daemon and let the spec do the orchestration.

If you need a hosted lifecycle today, use the TypeScript SDK directly or call any SDK's daemon over HTTP.

## Reranker availability differs

The reranker contract is identical across SDKs, but the bundled implementations are not:

- TypeScript ships an in-process cross-encoder, an LLM reranker, an HTTP / TEI reranker, and an Auto reranker that prefers a primary and falls back to a secondary.
- Go ships an HTTP / TEI reranker, an LLM reranker, and an Auto reranker. There is no in-process cross-encoder.
- Python ships an HTTP / TEI reranker, an LLM reranker, and an Auto reranker. There is no in-process cross-encoder.

For Go and Python users who want a cross-encoder today, point the HTTP reranker at a `bge-reranker-v2-m3` served by `llama-server` or another TEI-compatible endpoint.

## CLI parity is narrower

Library parity and daemon parity are ahead of CLI parity.

- TypeScript carries the broad local CLI surface.
- Go's production-ready CLI commands today are `version`, `serve`, and `eval lme`.
- Python's production-ready CLI command today is `serve`.

For cross-SDK automation, target the HTTP daemon or MCP tools rather than the
local CLI.
